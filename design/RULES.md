# design/ 目录规则

> 本目录记录"为什么这么干"的思考过程，不是方案文档的副本。
> 方案文档写"做什么"（`requirements/`），这里写"为什么"（`design/`）。

---

## 定位

`design/` 是思考沉淀层，记录开发过程中的决策推理。和方案文档、记忆文件的关系：

| 目录 | 定位 | 写什么 | 什么时候写 |
|------|------|--------|-----------|
| `requirements/` | 方案 | 做什么、怎么做 | 开发前 |
| `design/` | 思考 | 为什么这么做、为什么不那么做 | 开发中/后 |
| `.claude/memory/` | 知识 | 已确认的事实、踩过的坑、架构状态 | 每次会话 |

三者不重复：
- `requirements/` 有结论就够了，不需要写推理过程
- `.claude/memory/` 记事实和教训，不需要写完整的思考链路
- `design/` 把"想的过程"留下来

---

## 能写什么

### 必须写（有则记，没有不强求）

1. **"为什么"决策**：选 A 不选 B 的原因，尤其是 B 看似更优但最终没选的
2. **"不做"的决策**：不做什么比做什么更难回忆，"不做"的理由比"做"的理由更有记录价值
3. **决策演进**：如果先想了简单方案 → 后面改进了，用 `> **演进（日期）**` 追加，不要覆盖原内容
4. **跨模块通用原则**：从多个模块开发中渐进浮现的认知，不属于单一模块但指导未来开发

### 可以写

5. **命名讨论**：为什么叫这个名字不叫那个名字（如 `ack_cleanup` vs `release` vs `ack`）
6. **方案对比**：考虑过的多个方案及其取舍
7. **第三方库行为验证**：读源码确认的库能力边界（如 picohttpparser HTTP/1.x 限制）

### 不写

8. **方案细节**：那是 `requirements/` 的事
9. **事实清单**：那是 `.claude/memory/` 的事
10. **调试过程**：那是 `module_dev_experience.md` 的教训
11. **进度追踪**：那是 `STATUS.md` 的事

## 怎么记

### 文件命名

```
design/<模块或主题>-rationale.md
```

示例：
- `bus-split-rationale.md` — bus 拆分重构的决策
- `facade-plugin-architecture-rationale.md` — Facade+Plugin 架构的决策
- `http-implementation-rationale.md` — HTTP 实施阶段的决策

### 条目格式

```markdown
## 决策 N：简短标题

### 问题

遇到了什么问题/需要做什么决定？

### 决策

最终决定是什么？

### 原因

为什么这么做？为什么不做替代方案？

> **演进（日期）**：如果后续有变化，在这里追加。说明为什么改、改了什么、新方案好在哪。
```

### 演进追加规则

当决策发生变化时，**追加不覆盖**：

```markdown
### 决策 2：xxx

**决策**：方案 A

**原因**：...

> **演进（2026-07-01）**：改为方案 B
>
> 原方案 A 的问题：...
>
> 方案 B 的改进：...
>
> 原则提炼：...
```

保留原内容的原因：思考是渐进的，先想简单的再改进是正常的思考过程，这个轨迹本身有价值。

## 流程

### 模块开发完成后

用户发出指令（如"沉淀 design"），Claude 执行以下步骤：

1. **回顾本次模块开发的讨论和决策**
   - 从对话历史中找出"为什么"类讨论
   - 从 `module_dev_experience.md` 中找出教训背后的决策推理
   - 从 `.claude/memory/` 中找出跨模块渐进浮现的原则

2. **判断归属**
   - 属于特定模块 → 追加到该模块的 rationale 文件
   - 跨模块通用 → 追加到已有的跨模块文件，或新建主题文件
   - 已有条目发生变化 → 用演进格式追加，不覆盖

3. **检查过时内容**
   - 如果已有条目的结论与最终实现不一致，追加演进说明
   - 不修改原内容

4. **去重检查**
   - 不重复 `requirements/` 中已有的方案细节
   - 不重复 `.claude/memory/` 中已有的事实清单
   - 专注"为什么"——方案和记忆文件不覆盖的思考链路

### 检查清单

- [ ] 每个"为什么"都有完整的推理链路（问题 → 考虑过什么 → 为什么选这个）
- [ ] "不做"的决策有记录（不做什么 + 为什么不做）
- [ ] 变更的决策有演进说明（不改原内容）
- [ ] 没有重复 `requirements/` 或 `.claude/memory/` 的内容
- [ ] 跨模块原则已提炼（如果适用）
